# 消息顺序、偏序与拓扑

> 本文是「消息如何被编排、生成、演出」的权威说明。核心结论:**生成顺序 ≠ 演出顺序**,
> 演出顺序是一张**气泡偏序图(DAG)的拓扑排序**。理解这一点,交错与打断就都不再是特例。

---

## 1. 概念分层(先钉词汇)

领域层的 `Message` = 一条**气泡**(有稳定 id、是回复锚点)。围绕它有四个层级,不要混用:

| 层级 | 含义 | 时钟 |
| --- | --- | --- |
| **Conversation 对话** | 整个房间的对话流 | — |
| **Beat 节拍** | 上层一次**编排**决定的单元,可含多个说话者(交错、打断都发生在一个 beat 内) | **宏观时钟**(beat 之间的间隔) |
| **Turn 回合** | 一个角色的一次生成动作 → 产出若干气泡 | — |
| **Bubble 气泡 = 域 `Message`** | 原子发送单元,一条 Telegram 消息,稳定 id,回复锚点 | **微观时钟**(气泡级 typing / 停顿 / 迟疑) |

层级关系:一个 beat 含 ≥1 个 turn;一个 turn 含 ≥1 个气泡。

## 2. 两个时钟

调度的"时间"其实是两套,分属两个模块,**不要糊在一起**(今天 orchestrator 用两个常量把它们混了):

- **宏观时钟** —— beat 与 beat 之间隔多久、这一拍要不要留白。属于 **Conductor(编导)**。
- **微观时钟** —— 一个 turn 内气泡之间的 typing 时长、停顿、迟疑抖动、打断切入点。属于 **Delivery(演出)**。

"打字速度 → 输入间隔"是微观时钟,由角色的 typing 速度 + 文本长度 + 模型 hint 共同推出。

## 3. 生成顺序 ≠ 演出顺序(全篇的地基)

> A 说 A、B、C 三样;B 说 1、2、3 三样;最终播成 **A 1 2 B 3 C**。

要做到这个,**生成产出气泡的顺序**必须和**演出播放气泡的顺序**解耦:

- 生成可能先出全 A(A,B,C)再出全 B(1,2,3);
- 演出层再按编排重排成 A,1,2,B,3,C。

这条解耦是交错与打断能力的唯一结构前提。一旦承认它,"谁先谁后"就不再是一条直线,而是一组**约束**。

## 4. 全序与偏序(严格定义)

**全序(total order)**:集合里**任意两个元素都可比**。关系 `≤` 满足:

- 自反:`a ≤ a`
- 反对称:`a ≤ b` 且 `b ≤ a` ⟹ `a = b`
- 传递:`a ≤ b` 且 `b ≤ c` ⟹ `a ≤ c`
- **全性**:`∀ a,b`,要么 `a ≤ b` 要么 `b ≤ a`

结果是**一条直线**,每个元素位置唯一。普通聊天记录就是全序:消息 1、2、3、4,任意两条谁先谁后都定死。

**偏序(partial order)**:满足自反、反对称、传递,**但不要求全性**——允许一对元素 `a`、`b`
**既非 `a ≤ b` 也非 `b ≤ a`**,称它俩**不可比(incomparable)**,即"谁先谁后不受约束"。

**全序是偏序的特例**:当每一对元素都可比(全性成立),偏序就退化成全序。

## 5. 气泡图 BubbleGraph:把偏序画成 DAG

偏序天然对应一张 **DAG(有向无环图)**:节点是气泡,**边 `x → y` 读作"x 必须排在 y 之前"**。

两类边:

- **turn 内部边(必然全序)**:同一个人一个 turn 里的气泡内部有序 —— `A → B → C`。
  你不会把一个人自己一句话的顺序打乱。
- **跨说话者边(因果 / 回复)**:仅当存在依赖时才连。若 `1` 回应 `A`,加一条 `A → 1`。
  没有这类边时,跨人的气泡**默认不可比**,演出层可自由交错取质感。

**演出顺序 = 这张 DAG 的一个拓扑排序(topological sort)** —— 任何不违反箭头方向的线性化都合法。

### 例:A 1 2 B 3 C

```
turn 内部边:  A → B → C        （A 的三句自身有序）
              1 → 2 → 3        （B 的三句自身有序）
跨人边(若 1 咬 A): A → 1

合法拓扑排序举例:
  A, 1, 2, B, 3, C     ← 交错
  A, B, C, 1, 2, 3     ← 不交错,也合法
  1, 2, 3, A, B, C     ← 若无 A→1 边则合法;有该边则非法(违反 A 在 1 前)
```

**全序退化**:若 storyteller 想要严格"一句接一句",就把边加满,让 DAG 退化成单链,拓扑排序唯一 ——
此时没有交错自由。**要交错就少加边(留不可比),要严格就加满边(退化全序)。**

## 6. 打断的两种类型(本质不同)

### 6.1 AI ↔ AI 打断 = 剧本化(scripted),常态下不发生

storyteller / Conductor **在生成前**就决定这一桥段有打断。于是:

- 所有 generator 带着**预知**生成:被打断者写出"读到 B 就被截也通顺"的内容;插话者带着对被切内容的预知写 barge-in。
- **没有鸡生蛋、没有真重排** —— "重排"只是演出层对一批**彼此知情、已生成**气泡的叙事编排。

在数据上,剧本化打断 = **Conductor 在生成前把边写死**:插话气泡 → 被打断者剩余气泡(表达"截断/后置"),
被截的气泡在 beat plan 里标为**条件丢弃或收尾成 trailing-off**。**边是作者写的(authored),不是 live 发现的(discovered)。**

> 剧本化 ≠ 稀有:Conductor(高频轻量)也可基于张力读数顺手编一个廉价 barge-in,让冷场/僵持时有人"忍不住插一句",
> 既保留可控,又有涌现感,不必每次惊动重量级 storyteller。

### 6.2 用户 → AI 打断 = 真·反应式(reactive),**强制实现**

用户何时插话、扔什么,storyteller 无法预知,**不可能剧本化**。这是**唯一需要真中断机制**的入口。
AI 侧永远照剧本演,不需要 live 抢占;真抢占只被用户输入触发,复杂度因此收窄到一个触发点。

## 7. 用户打断机制

1. **判断器(ingest 时的判据)**:给用户输入分类 —— **附和** vs **关键打断**
   (是 M2 意图分类"推进/捣乱/试探/拐弯"在"要不要抢占"维度上的投影)。便宜模型或规则,与 Conductor 同级的暗线判据。
2. **附和** → 不抢占:摄入共享历史,当前 beat 演完,下一次 Conductor 规划自然看到它。
3. **关键打断** → 抢占:
   - 通知 Delivery **清空当前 beat 尚未发出的气泡**(丢弃,或改成"话说一半 trailing off"的叙事收尾);
   - 在飞的生成(线程池里还在产气泡的 turn):**MVP 最简做法是让它跑完但丢弃结果**(标 `beat_aborted`),
     贵了再上 cancellation token;
   - Conductor 立即规划一个**回应用户输入**的新 beat。

> **对称**:storyteller 的"僵局时抛压力"与用户的"僵局时抛炸弹"是同一种动作 —— 一个排期、一个反应。
> 两者汇进**同一个"打断 / 重规划"触发口**,Delivery 与 Conductor 不必区分炸弹来自谁。

## 8. 不变式(改动时必须守住)

- **因果偏序不可违反**:若 `y` 依赖 `x`(回复 / 反应 / turn 内后继),演出顺序里 `x` 必在 `y` 前。
  重排是"挑一个合法拓扑排序",不是自由打乱。
- **单消费者队列**:延续"只有 `gateway.complete` 下线程池"的并发不变式 —— 生成在线程池里**只往队列 append**(带偏序边),
  Delivery 在 **event loop 线程**单消费。抢占 = loop 线程"清 pending + 置 `beat_aborted`",生成线程的产物被丢弃。
  多写单读,无锁竞争。
- **两平面**:前台消息流**不碰 tool**;模型知道的"乐器"是 **marker 词表**(见 §9),不是 function-calling。

## 9. 权威分工(谁决定什么)

- **Conductor 只定结构**:谁上场、谁打断谁、走位 —— 即**作者这张 DAG(哪些边存在)**。
- **模型只填内容与质感**:说什么、多迟疑、想不想插嘴。它可以**请求** barge-in(emit marker),但批不批是 Conductor 的。
  模型的意见通过 **marker(模型 emit / Delivery 消费 的唯一通道)** 表达:`{{SEPARATOR}}` / `{{JITTER}}` /
  `{{HESITATE:0.5}}` / `{{BARGE}}` …… **模型提议,Delivery 裁决(model proposes, delivery disposes)。**
- **Delivery 只演出**:把模型 hint + 角色 factor(typing 速度 / 迟疑倾向,存角色卡)+ 全局约束调和成最终时序,
  并挑一个合法拓扑排序播出。

一句话:**Conductor 作者 DAG,模型填血肉,Delivery 挑拓扑排序去演。** 三个权威互不越界 —— 这就是"职责清晰"的定义。
